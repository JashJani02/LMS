# SKILL: DOCUMENTATION

## When to use
- Writing README.md
- Creating guides or setup instructions
- Explaining projects or features

## Rules
- ALWAYS be clear and structured
- ALWAYS use markdown formatting
- ALWAYS include headings and sections
- Keep language simple and beginner-friendly

## Structure
1. Title
2. Description
3. Features
4. Installation
5. Usage
6. Example (if applicable)
7. Contributing (optional)

## Output
- Clean markdown only
- No unnecessary explanations outside the doc
